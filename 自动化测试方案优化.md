# 自动化测试方案优化

## 一、现状问题

### ui-auto-test 当前局限

1. **无联动测试能力**
   - 仅做爬虫式页面扫描，无法模拟多步骤流程
   - 无法保持登录态、Cookie、LocalStorage 跨页面传递
   
2. **无业务逻辑感知**
   - 不理解后端 API 业务流程
   - 无法针对边界条件、异常处理设计测试用例

3. **无覆盖率评估**
   - 跑完不知道覆盖了多少场景
   - 不知道哪些关键路径还没测到

---

## 二、整合方案

### 核心思路

结合三个 skill 的优势：
- **coding-agent** → 代码分析 + 测试用例生成
- **ui-auto-test** → 批量错误检测（headless 模式）
- **computer-use** → 关键流程验证（有界面 + 视频录制）

### 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    integrated-test                      │
├─────────────────────────────────────────────────────────┤
│  1. 分析阶段                                            │
│     ├── 扫描后端 API 路由 → 生成待测清单                  │
│     ├── 分析业务代码 → 提取边界条件、异常处理              │
│     ├── 生成业务流程图 → 识别关键路径                     │
│     └── 输出：场景清单 + 覆盖率基线                       │
├─────────────────────────────────────────────────────────┤
│  2. 生成阶段                                            │
│     ├── 根据 API 清单生成基础用例                         │
│     ├── 根据边界条件补充异常用例                          │
│     ├── 根据业务流程生成联动用例                          │
│     └── 输出：测试用例集（Playwright 脚本）               │
├─────────────────────────────────────────────────────────┤
│  3. 执行阶段                                            │
│     ├── Phase 1: headless 批量扫描                       │
│     │   └── 快速检测 JS 错误、网络失败、白屏等             │
│     └── Phase 2: 有界面执行关键流程                      │
│         └── 录制视频、捕捉视觉问题、验证复杂交互           │
├─────────────────────────────────────────────────────────┤
│  4. 报告阶段                                            │
│     ├── 错误汇总 + 严重程度分级                          │
│     ├── 覆盖率报告（API 覆盖、流程覆盖）                  │
│     ├── 视频/截图证据                                    │
│     └── 建议：哪些场景需要补充                           │
└─────────────────────────────────────────────────────────┘
```

## 三、覆盖率评估方法

### 1. API 覆盖率

```typescript
// 扫描所有路由
const routes = scanRoutes('./src/routes'); // API 端点清单
const tested = extractFromTestCases(tests); // 已测试的 API
const coverage = tested.size / routes.size;
const missing = routes.filter(r => !tested.has(r));
```

### 2. 代码覆盖率（可选）

- 后端：使用 Istanbul/JaCoCo
- 前端：使用覆盖率报告

### 3. 流程覆盖率

```
业务流程:
  登录 → 浏览商品 → 加入购物车 → 结算 → 支付 → 订单成功
  
已覆盖路径:
  ✓ 登录 → 浏览商品 → 加入购物车
  ✓ 登录 → 结算 → 支付（未登录跳转）
  ✗ 支付失败后重试
  ✗ 购物车跨设备同步
```

## 五、使用方式

```bash
# 启动测试（自动分析 + 生成 + 执行）
integrated-test --project /path/to/project --url http://localhost:3000

# 仅分析（不执行测试）
integrated-test --project /path/to/project --analyze-only

# 仅生成用例（不执行）
integrated-test --project /path/to/project --generate-only
```
