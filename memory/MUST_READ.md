## 🤖 多 Agent 协作规则（重要！）

### 会议模式（智能启用）
- **默认状态**: 单模型模式，快速响应
- **自动恢复**: 有进行中的复杂任务时，自动恢复双模型并提示

**复杂度判定提示**（每次判定后在标题括号中显示）：
- 【复杂问题】→ 开启会议模式，双模型讨论
- 【简单问题】→ 单模型快速响应

**复杂度判定标准**（满足任一即判定为复杂）：
1. 多步骤任务（步骤1/2/3、阶段、流程）
2. 涉及方案对比（方案A/B、对比、优劣、权衡）
3. 需要风险评估（风险、问题、故障、隐患）
4. 需要决策（选择、决定、是否、建议）
5. 涉及架构/设计（架构、设计、方案、优化）

**快速触发词**（出现任一即触发）：
- 显式：讨论、会议、你怎么看、一起分析、协作者
- 隐式：评估、方案对比、决策、优化建议、风险分析、设计、架构、选型

**快速模式关键词**（出现任一即跳过）：
- 直接回答、跳过讨论、非会议模式、快速回复、简要说明、一句话

**MiniMax 超时设置**:
- 等待时间：最长 **10分钟**（600秒）
- 检测间隔：每30秒检查响应状态
- 超时处理：记录超时，使用主 Agent 方案继续

**复杂度自动检测**（满足2项建议开启）：
1. 多步骤任务（步骤1/2/3、阶段、流程）
2. 涉及方案对比（方案A/B、对比、优劣、权衡）
3. 需要风险评估（风险、问题、故障、隐患）
4. 需要决策（选择、决定、是否、建议）
5. 多维度分析（技术/成本/时间/维护）

- **规则**:
  1. 双向提案：双方独立提出方案，不盲从
  2. 透明讨论：所有内容对用户可见
  3. 执行监督：非执行方负责评估和偏差告警
  4. 及时纠正：发现问题立即提出

### 协作者 Agent 配置
- **标签**: `minimax-native`
- **模型**: MiniMax-M2.1（通过转接）
- **API**: https://nexusacc.itssx.com/api/claude_code/cc_minimax21
- **职责**: 独立提案、双向质疑、建设性融合
- **触发方式**: 自动恢复（有任务时）或快速触发词

### 执行阶段规则
- **执行前**: 明确执行方案、预期结果、验收标准
- **执行中**: 实时同步进度，非执行方监督
- **执行后**: 对比预期 vs 实际，评估偏差
- **偏差处理**: 
  - 轻微偏差：执行方调整
  - 严重偏差：暂停执行，重新讨论
  - 失败：分析原因，启动备选方案
